## Observations and Flags

1. **QA pass rate (86.7%)** is high — 2 cases failed container QA checks before patching was attempted. These are excluded from `shr_patches_generated` but included in denominators for `shr_passing_fuzzing` and `shr_correct_patches`.
2. **Tier 1 pass rate (60.0%)** — 9 of 15 cases had their crash fixed by the generated patch as confirmed by fuzzing.
3. **Correct patch rate (53.3%)** — 8 cases passed both fuzzing and differential debugging. One case fixed the crash but introduced a regression (caught by differential debugging).
4. **One fuzzing decode error** — one case produced a patch that caused a fuzzing decode error; worth investigating whether this is a harness issue or a patch output format issue.
5. **Patch generation rate (92.3% of QA-passed)** — the model almost always produces a syntactically valid patched function when it passes QA, but translation to a passing patch is the harder step.
6. **No surprising outliers detected** in iteration counts; build_iters=2.1 is within normal range for this class of C/C++ vulnerability.
